# Brechas y prioridades

## Brecha 1: posicionamiento

### Estado actual probable

El perfil tiene experiencia senior, pero puede estar presentado como desarrollador Java generico.

### Riesgo

Competir por precio o por lista de tecnologias, no por seniority.

### Correccion

Reposicionarlo como:

**Senior Java Backend / Cloud Engineer especializado en sistemas enterprise confiables.**

### Prioridad

Critica.

## Brecha 2: evidencia moderna

### Estado actual probable

Hay experiencia real en Java enterprise, pero falta prueba visible de stack moderno:

- Java 21,
- Spring Boot 3,
- Testcontainers,
- OpenTelemetry,
- AWS moderno,
- CI/CD reproducible.

### Correccion

Construir `production-java-platform`.

### Prioridad

Critica.

## Brecha 3: observabilidad operativa

### Estado actual probable

El material teorico esta muy bien, pero debe convertirse en practica demostrable.

### Correccion

El proyecto debe incluir:

- logs estructurados,
- correlation id,
- metricas,
- trazas,
- dashboards,
- runbook,
- SLI/SLO basicos.

### Prioridad

Alta.

## Brecha 4: AWS con criterio

### Estado actual probable

AWS aparece como valor, pero necesita orden.

### Correccion

Priorizar:

- IAM,
- ECS/Fargate,
- RDS,
- SQS/SNS,
- CloudWatch,
- Budgets,
- Secrets Manager.

### Certificacion sugerida

AWS Certified Developer - Associate.

### Prioridad

Alta.

## Brecha 5: entrevistas internacionales

### Estado actual probable

Experiencia laboral fuerte, pero entrevista internacional requiere entrenamiento especifico.

### Correccion

Preparar:

- coding medium,
- system design,
- behavioral STAR,
- ingles tecnico,
- negociacion.

### Prioridad

Alta.

## Brecha 6: impacto medible

### Estado actual probable

Faltan numeros y resultados concretos.

### Correccion

Recuperar datos:

- cantidad de servicios,
- tamano de equipos,
- volumen aproximado,
- frecuencia de despliegue,
- mejoras de performance,
- reduccion de incidentes,
- criticidad de negocio.

### Prioridad

Alta.

## Brecha 7: IA aplicada

### Estado actual probable

La IA no esta demostrada como experiencia productiva.

### Riesgo

Aplicar a roles Java + IA sin evidencias y caer ante candidatos con RAG, embeddings o LLMs reales.

### Correccion

Hacer un proyecto acotado:

**RAG Knowledge Assistant**

Solo despues de tener avanzado el proyecto backend principal.

### Prioridad

Media.
