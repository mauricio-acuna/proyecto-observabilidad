# Analisis de ficheros

## Proposito

Este documento responde tres preguntas:

1. Que ficheros originales sirven para el plan 2027.
2. Que informacion sobra o debe quedar como secundaria.
3. Que informacion falta para que el proyecto sea ejecutable.

## Veredicto general

Todos los ficheros pueden aportar algo, pero no todos deben tener el mismo peso.

El plan 2027 debe tomar como nucleo:

- perfil profesional real,
- Java moderno,
- Spring Boot,
- AWS/cloud,
- observabilidad,
- operacion,
- entrevistas,
- mercado internacional.

Debe dejar como secundario:

- IA como identidad principal,
- mobile,
- diplomaturas generalistas,
- side hustles,
- tecnologia emergente sin relacion directa con empleo senior.

## Que entra al plan

### Entra como nucleo

- Java 17/21.
- Spring Boot 3.
- APIs REST bien disenadas.
- PostgreSQL.
- Testing serio.
- Docker.
- CI/CD.
- AWS.
- Microservicios razonables.
- Mensajeria.
- Observabilidad.
- Resiliencia.
- Seguridad.
- Ingles tecnico.
- System design.
- Coding interview.
- CV y LinkedIn internacionales.
- Networking y aplicaciones.

### Entra como complemento

- Kubernetes.
- GitOps.
- Platform Engineering.
- Virtual Threads.
- Spring Modulith.
- Spring AI.
- GraalVM.
- IA aplicada a backend.
- MCP/tool orchestration.

Estos temas sirven si no compiten con las prioridades principales.

## Que sobra o debe quedar fuera

### Fuera del eje principal

- Mobile como cambio profesional.
- IoT como camino principal.
- Ebooks o ingresos pasivos como estrategia central.
- Diplomaturas generalistas de IA.
- Bootcamps fullstack.
- Reinvencion como AI Engineer.
- Cursos largos sin proyecto demostrable.
- Tecnologias emergentes usadas solo por ansiedad.

### No se eliminan como ideas

Pueden quedar como curiosidad o futuro, pero no deben consumir energia del proyecto 2027.

## Que falta

### Falta informacion del perfil real

- CV actual final.
- LinkedIn actual.
- Nivel real de ingles probado.
- Salario actual y salario minimo aceptable.
- Disponibilidad semanal real.
- Restricciones familiares, energia y horarios.
- Preferencia remoto/hibrido/presencial.
- Lista de empresas objetivo.

### Falta evidencia concreta

- Proyectos publicos.
- Repositorios.
- README profesional.
- Diagramas.
- Certificaciones.
- Resultados medibles en trabajos anteriores.
- Historias STAR con datos.
- Metricas de impacto.

### Falta sistema de ejecucion

- Seguimiento semanal.
- Backlog priorizado.
- Calendario de aplicaciones.
- Registro de entrevistas.
- Registro de feedback.
- Registro de empresas.
- Revision mensual.

## Decision final de filtrado

El proyecto 2027 debe usar todo el material existente, pero con jerarquia:

1. Perfil y mercado primero.
2. Backend/cloud/observabilidad segundo.
3. Entrevistas y aplicaciones tercero.
4. IA solo como complemento.
5. Temas laterales fuera del camino critico.
