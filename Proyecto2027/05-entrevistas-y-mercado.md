# Entrevistas y mercado

## Objetivo

Preparar a Mauricio para pasar procesos senior internacionales, no solo entrevistas locales de Java.

## Coding interview

### Estructura verbal

1. Let me restate the problem.
2. What are the constraints?
3. Can I assume the input fits in memory?
4. I will start with a brute-force approach.
5. We can optimize this using...
6. The time complexity is...
7. Let me test it with edge cases.

### Temas prioritarios

- Arrays.
- Strings.
- Hash maps.
- Intervals.
- Heaps.
- Trees.
- Graphs.
- Binary search.
- Dynamic programming basica.

### Objetivo antes de enero 2027

- 150-250 problemas acumulados.
- Resolver en voz alta.
- Explicar trade-offs.
- Practicar en ingles.

## System design interview

### Apertura recomendada

> Let me clarify the requirements first. I will separate functional and non-functional requirements, then propose a high-level architecture, discuss data models and APIs, and finally go deeper into scalability, reliability, security, observability and trade-offs.

### Checklist

- Requirements.
- Traffic estimation.
- APIs.
- Data model.
- Components.
- Storage.
- Cache.
- Queue/events.
- Scaling.
- Consistency.
- Failure handling.
- Security.
- Observability.
- Cost.

### Diseños a practicar

- URL shortener.
- Distributed queue.
- Job scheduler.
- Payment processing system.
- Audit logging platform.
- Notification system.
- Monitoring platform.
- Feature flag system.
- Distributed rate limiter.
- Banking transaction monitor.
