Dolores Principales

Lo que no se debe perder de vista antes de vender estos proyectos como productivos.

Persistencia real pendiente

Varios proyectos usan adapters in-memory. Falta PostgreSQL, Flyway y Testcontainers donde aplique.

Eventos reales pendientes

El hub de eventos modela idempotencia, pero falta broker real, retries persistentes y DLQ observable.

Observabilidad por validar

Hay collector, Tempo, Prometheus y Grafana versionados. Falta ejecutar Docker y confirmar trazas end-to-end.

Performance no ejecutada

Existen scripts k6 para APIs criticas. Aun falta levantar servicios y capturar baseline de latencia p95.
